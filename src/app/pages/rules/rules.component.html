Rules

It is a two player game, each player starts with a planet of some size from which the player can send his fleet to other planets.

A player can only send half of the fleet available at a given planet.

A player can take ownership of another planet by sending a larger fleet then the fleet defending the planet.

Every turn new spaceships are produced and added to the planet's fleet depending on the planet's size.

Attacking your own planet is considered a reinforcment.

The game is over when:

  • A player's entire fleet is assimilated by the collective (run out of fleets and planets)
  • A player takes more then 100ms to produce a move
  • A player's move is invalid
  • A player decides to abandon the cluster, and run (surrender)
  • A player needs more moves then the size (x*y) of the cluster (game takes too long - timeout)

How to participate

The competition is structured into two phases, namely development phase, and tournament phase. In the development phase an automatic hidden tournament is available. To join the development phase and thus the turnament, login and click the Join button at the top of this page (if you didn't already). Perioidcally, all submitted players will play a set of games and results will be displayed for your submission. To submit your player you must upload a .zip archive containing only your source .java files.
The submitted project must contain a java file Igralec.java, whith a main function. Files within your archive will be extracted and compiled using Java JDK 11. The downloadable Game Pack contains Evaluator.jar, Gui.jar, and a sample player Igralec.java.
After the development phase ends, a tournament will be announced in which players will be ranked by playing amongst each other.

Instructions for running the game

The game is ran by moving into the directory you downloaded Evaluator.jar to (all the following commands will be run from this folder). First you need to create the two forlders for the players you want to match (each player should be in its own folder). For this example we will create folders called TestPlayer and MyPlayer 

mkdir TestPlayer MyPlayer

After creating the folders we will move the Igralec.java (random player) into both of the folders, by running commands: 

cp Igralec.java MyPlayer
cp Igralec.java TestPlayer

Once both folders contain the code for the random player, we will compile both players, to obtain the *.class files. 

javac MyPlayer/*.java
javac TestPlayer/*.java

The game is ran by running Evaluator.jar. This is "the game" jar. 

java -jar Evaluator.jar SpaceBattleship <player1> <player2>
where player1, and player2 are folders with compiled players.

As an example, if you moved the downloaded player into the folder TestPlayer and the player that contains the code you are developing into the folder MyPlayer, you run the game as: 

java -jar Evaluator.jar SpaceBattleship TestPlayer MyPlayer

If you also want to draw the game real-time, the Game_pack contains Gui.jar, that you can use. You run the game with GUI by running the following command: 

java -jar Evaluator.jar SpaceBattleship <Player1> <Player2> | java -jar Gui.jar

Example directory structure: 

            dir
            |--Evaluator.jar
            |--Gui.jar
            |--TestPlayer
            |     |--Igralec.class
            |
            |--MyPlayer
            |     |--Igralec.class

Instructions for creating a player

A player must be a standard java program, that uses the standard input and standard output to communicate with the game.
The player's main method must be in the file named Igralec.java - without the use of packages, jars, etc.
A player MUST NOT:
  • use the network
  • access (not even read) the filesystem outside of its directory
  • run system commands
  • try to win the game by any other means then the commands outlined below
  • be a copy of another player :)
  • The above lines will be filled if the need arises and will be enforced retroactively
Commands that a player sends to the game are in the form: 
<command> <parameters>

All available commands:

command parameters description sender
U <int>,<int>,<string> Universe - size (x in y) playing field (allways 100,100), and your color game
P <int>,<int>,<int>,<float>,<int>,<string> Planet:
- name (number),
- position x,
- position y,
- planet size,
- fleet size,
- planet color (red, blue or null)		 game
F <int>,<int>,<int>,<int>,<int>,<int> Fleet:
- flee name (number),
- fleet size
- origin planet
- destination planet
- current turn
- number of needed turns 		game
S - Start - when a player receives this command from the game, it must answer with a series of A, followed by an E game
A <int>,<int> Attack:
- origin planet
- destination planet		 player
E - Player sends at the end of an attack player
L <String> Player can send this command to forfeit the game, the parameters are displayed as the reason for forfeiting game, player
